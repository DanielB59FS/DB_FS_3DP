45 //! A Proxy class that can be used to record multiple Events for later inspection.
46 /*!
47 * Receives and stores all events produced by an interface derived from GEventGenerator.
48 * You must provide the EventQueue a maximum storage capcity as the underying container is a circular quque.
49 * Accepts an std::function<void()> as the callback function/lambda for flexibility.
50 * Use std::bind to supply your own arguments to your callback, the only restriction is the void return type.
51 * Because all events are captured (assuming adequate storage) you may process events later when ready to do so.
52 * \code{.cpp}
53 * MyEventQueue.Create( 16, MyEventGenerator, nullptr); // if you only need the last event, use GEventResponder
54 * GEvent g;
55 * GEventGenerator::Events e; // replace with desired Generator
56 * GEventGenerator::EVENT_DATA d;
57 * while(+MyEventQueue.Pop(g)) // later, possibly in main loop...
58 * if(+g.Read(e,d))
59 * if(e == GEventGenerator::Events::NON_EVENT) // replace with actual event
60 * std::cout << d.noData; // replace with actual data
61 * \endcode
62 * \deprecated **Replaced by GW::CORE::GEventCache.**
63 */

93 //! Allocates & Intializes a GEventQueue.
94 /*!
95 * A GEventQueue is capable of receiving and reacting to events created by a GEventGenerator.
96 * You can use this class whenever there is a Gateware class that spawns events you wish to know about.
97 * Unlike GEventReceiver the GEventQueue is capable of storing multiple events up to a limit defined by you.
98 * If you need to process events on a specific thread, you can use this class to delay processing until
99 * a safe time to do so. Just make sure there is adequate capcity in the Queue, and "Pop" them off when ready.
100 *
101 * \param [in] _maxSize The maximum amount of messages that can be retained by this instance. (must be > 0)
102 * Exceeding this amount without processing a message will cause the oldest to be missed.
103 * \param [in] _eventSource The GEventGenerator you wish to receive messages from.
104 * \param [in] _callback The function or lambda you wish to be invoked when an event is received.
105 * (optional, can pass nullptr). May be invoked from a different thread.
106 * To ensure you receive messages on a specific thread, just use "Pop/Peek" instead.
107 *
108 * \retval GReturn::UNEXPECTED_RESULT Internal API failure, should not happen please report this.
109 * \retval GReturn::INVALID_ARGUMENT A valid GEventGenerator and Queue Capacity > 0 must be provided.
110 * \retval GReturn::SUCCESS The receiver was correctly bound and registered to the generator provided.
111 */
112 GReturn Create(unsigned int _maxSize, CORE::GEventGenerator _eventSource, const std::function<void()> _callback);
113
114 //! Examines but does not remove a waiting event if one is available based on it's historical index.
115 /*!
116 * Same operation as Peek() in GEventReceiver but can also be used to examine the Queue's current backlog.
117 *
118 * \param [in] _eventIndex The event you wish to examine from 0 to "Waiting"-1, with 0 being the most recent.
119 * \param [out] _outEvent A copy of the event at _eventIndex waiting it's turn to be processed.
120 *
121 * \retval GReturn::INVALID_ARGUMENT The provided _eventIndex was out of bounds.
122 * \retval GReturn::FAILURE There were no waiting events that could be inspected.
123 * \retval GReturn::SUCCESS Successfully copied a waiting event from the Queue.
124 */
125 virtual GReturn Peek(unsigned int _eventIndex, GEvent& _outEvent) const = 0;

132 *
133 * \param [out] _outCapacity The maximum capacity of the Event Queue.
134 *
135 * \retval GReturn::SUCCESS The maximum capacity was successfully reported.
