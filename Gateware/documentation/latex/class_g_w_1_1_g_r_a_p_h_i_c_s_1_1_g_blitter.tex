Loads and stores a read-\/only raster source to define tiles from. 

Use this function to load source data from files directly. All types of data are optional, but at least one must be used for a source to be valid. All images used must have the same dimensions. Greyscale (especially with no alpha channel) is recommended for layer/stencil data since the filesize will be significantly smaller. If a file does not contain an alpha channel, the alpha values will be set to maximum. Supported file types\+: TGA (Targa) Supported color formats\+:
\begin{DoxyItemize}
\item 8 bits-\/per-\/pixel greyscale (Luminance) \mbox{[}TGA documentation refers to this as black-\/and-\/white\mbox{]}
\item 16 bits-\/per-\/pixel greyscale with transparency (Alpha-\/\+Luminance) \mbox{[}TGA documentation refers to this as black-\/and-\/white\mbox{]}
\item 24 bits-\/per-\/pixel RGB color (Red-\/\+Green-\/\+Blue) \mbox{[}TGA documentation refers to this as true-\/color\mbox{]}
\item 32 bits-\/per-\/pixel RGB color with transparency (Alpha-\/\+Red-\/\+Green-\/\+Blue) \mbox{[}TGA documentation refers to this as true-\/color\mbox{]}
\item 8 bits-\/per-\/pixel indexed color (Look-\/\+Up Table) \mbox{[}TGA documentation refers to this as color-\/mapped\mbox{]} -- Only 24-\/bit palette entries and 8-\/bit palette indices are supported. -- Be sure all colors in the palette are opaque, otherwise the palette will be read incorrectly. Maximum dimensions supported\+: 65,535 x 65,535 (TGA files do not support anything larger than this, so attempting to export a TGA larger than this will corrupt the data. The library is not able to detect if this has happened except at exact multiples of 65,536 in either dimension.) Maximum sources supported\+: 65,535
\end{DoxyItemize}
